# 知识库文档分析 Skill

## 功能
读取 `knowledge-base/raw/` 目录中的原始文档，使用 LLM 进行分析、摘要和标签提取，生成结构化的知识页面到 `knowledge-base/wiki/` 目录。

## 使用场景
当用户说以下任意一句话时触发：
- "分析文档"
- "处理文档"
- "导入文档"
- "ingest documents"
- "分析 raw 目录"

## 执行步骤

### 1. 扫描原始文档
```bash
# 查看 raw 目录有哪些文档
ls -la knowledge-base/raw/
```

### 2. 读取文档内容
对于每个未处理的文档（检查 wiki 目录是否已存在对应页面）：
- 读取文档完整内容
- 识别文档类型（Markdown、TXT 等）

### 3. LLM 分析文档
对每个文档进行分析，提取：
- **标题**：从文档内容或文件名提取
- **摘要**：200字以内的核心内容总结
- **标签**：3-5个关键词标签
- **概念**：3-8个关键概念
- **分类**：确定文档所属的知识类别

分析提示词：
```
你是一个知识管理助手。请分析以下文档内容，并提供：
1. 一个简洁的摘要（200字以内）
2. 3-5个标签
3. 提取的关键概念（3-8个）
4. 文档分类建议

文档标题: {filename}
文档内容:
{content}

请以JSON格式返回：
{
  "title": "文档标题",
  "summary": "文档摘要",
  "tags": ["tag1", "tag2"],
  "concepts": ["concept1", "concept2"],
  "category": "分类名称"
}
```

### 4. 生成 Wiki 页面
为每个文档创建结构化的 Wiki 页面，保存到 `knowledge-base/wiki/articles/` 目录：

文件命名：`{概念或标题}.md`

内容格式：
```markdown
---
title: {标题}
summary: {摘要}
tags: [{标签列表}]
concepts: [{概念列表}]
category: {分类}
source: raw/{原文件名}
created: {当前时间}
---

# {标题}

## 摘要

{摘要}

## 核心内容

{原文关键内容，保持Markdown格式}

## 相关概念

{概念列表，使用双向链接格式 [[概念]]}

## 来源

- 原始文档: [{原文件名}](../../raw/{原文件名})
```

### 5. 更新索引
更新 `knowledge-base/wiki/INDEX.json` 文件：

```json
{
  "pages": [
    {
      "id": "{页面ID}",
      "title": "{标题}",
      "path": "articles/{文件名}.md",
      "summary": "{摘要}",
      "tags": ["标签列表"],
      "category": "{分类}"
    }
  ],
  "concepts": ["概念列表"],
  "categories": ["分类列表"],
  "lastUpdated": "{当前时间}"
}
```

### 6. 输出处理结果
向用户报告：
- 处理了多少文档
- 跳过了多少（已存在）
- 生成了哪些 Wiki 页面
- 遇到的错误（如果有）

## 注意事项
1. **增量处理**：只处理 raw 目录中新增的文档，跳过已处理的
2. **保持原文**：Wiki 页面应包含原文的核心内容，不要过度简化
3. **建立关联**：如果文档与已有 Wiki 页面相关，添加双向链接
4. **中文优先**：摘要和标签使用中文
5. **错误处理**：如果某个文档处理失败，记录错误但继续处理其他文档

## 示例输出
```
✅ 文档处理完成！

已处理: 3 个文档
已跳过: 2 个文档（已存在）

新增页面:
- 人工智能概述.md
- 机器学习基础.md
- 深度学习入门.md

请刷新浏览器查看更新后的知识库。
```
