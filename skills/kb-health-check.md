# 知识库健康检查 Skill

## 功能
全面检查知识库的质量，发现问题、不一致、缺失内容，并提供改进建议。

## 使用场景
当用户说以下任意一句话时触发：
- "健康检查"
- "检查知识库"
- "lint wiki"
- "质量检查"
- "知识库体检"

## 执行步骤

### 1. 扫描知识库结构
```bash
# 检查目录结构
tree knowledge-base/wiki/

# 检查索引文件
cat knowledge-base/wiki/INDEX.json

# 统计文件数量
ls knowledge-base/wiki/articles/ | wc -l
ls knowledge-base/raw/ | wc -l
```

### 2. 检查内容完整性

对每个 Wiki 页面检查：

#### 必须有
- [ ] YAML frontmatter（元数据）
- [ ] 标题（title）
- [ ] 摘要（summary）
- [ ] 标签（tags）
- [ ] 正文内容
- [ ] 来源引用

#### 应该有
- [ ] 相关概念链接
- [ ] 分类信息
- [ ] 创建/更新时间
- [ ] 双向链接

#### 问题记录
```
❌ 缺少摘要: 文章A.md
❌ 缺少标签: 文章B.md
⚠️ 没有引用来源: 文章C.md
⚠️ 缺少相关概念: 文章D.md
```

### 3. 检查链接有效性

#### 断裂链接检查
- 检查所有 `[[链接]]` 格式的内部链接
- 验证链接的目标文件是否存在
- 检查相对路径是否正确

```
❌ 断裂链接: [[机器学习]] -> 文件不存在
❌ 断裂链接: ../../raw/xxx.md -> 路径错误
```

#### 孤立页面检查
- 找出没有被其他页面链接的孤立文章
- 找出没有链接到其他文章的孤岛

```
⚠️ 孤立页面: 文章E.md（没有被任何其他页面引用）
⚠️ 孤岛页面: 文章F.md（没有引用任何其他页面）
```

### 4. 检查内容质量

#### 重复内容
- 比较文章间的相似度
- 找出重复或高度重叠的内容

```
⚠️ 重复内容: 
  - 文章A.md 和 文章B.md 相似度 85%
  - 建议: 合并或明确区分
```

#### 内容过时
- 检查长时间未更新的页面
- 标记可能需要更新的内容

```
⚠️ 可能过时: 文章G.md（最后更新: 30天前）
```

#### 格式一致性
- 检查 Markdown 格式是否统一
- 检查标题层级是否合理
- 检查代码块是否有语言标记

### 5. 检查知识结构

#### 分类平衡性
- 各分类的文章数量是否均衡
- 是否有分类过于拥挤或空旷

```
📊 分类统计:
- 机器学习: 15 篇
- 前端开发: 8 篇
- 后端开发: 2 篇 ⚠️ 较少
- 数据库: 1 篇 ⚠️ 较少
```

#### 知识深度
- 是否只有入门内容，缺少进阶
- 是否有高级内容但缺少基础铺垫

```
💡 建议:
- "深度学习"有 5 篇进阶文章，但缺少入门指南
- 建议创建"深度学习入门"
```

### 6. 生成健康报告

输出完整报告：

```markdown
# 知识库健康报告

## 总体评分: 85/100 ✅

### 📊 基础统计
- Wiki 页面: 26 篇
- 原始文档: 15 篇
- 概念数量: 32 个
- 平均链接数: 4.2 个/页

### ❌ 错误 (需要立即修复)

1. 断裂链接: 3 个
   - 文章A.md -> [[不存在的页面]]
   - 文章B.md -> 错误的路径
   - ...

2. 缺少必需元数据: 2 篇
   - 文章C.md 缺少摘要
   - 文章D.md 缺少标签

### ⚠️ 警告 (建议修复)

1. 孤立页面: 4 篇
   - 建议添加相关链接

2. 重复内容: 2 组
   - 文章E vs 文章F (85% 相似)

3. 格式不一致: 5 篇
   - 标题层级不统一
   - 代码块缺少语言标记

### 💡 改进建议

1. 内容补充
   - 为"数据库"分类添加更多文章
   - 创建"深度学习入门"引导文章

2. 结构优化
   - 将"前端"相关文章细分子分类
   - 为复杂主题创建知识图谱

3. 维护建议
   - 更新 3 篇超过 30 天未更新的文章
   - 合并 2 篇重复文章

### 📈 趋势

- 本周新增: 5 篇
- 本月更新: 12 篇
- 知识库健康度: 良好 ↑
```

### 7. 自动修复（可选）
如果用户同意，自动修复：
- 修正断裂链接（如果目标明确）
- 为缺失的页面添加基础元数据
- 统一格式问题
- 添加明显的双向链接

### 8. 更新索引
将检查结果写入 `knowledge-base/wiki/health-report.json`：

```json
{
  "score": 85,
  "errors": [...],
  "warnings": [...],
  "suggestions": [...],
  "lastChecked": "时间"
}
```

## 评分标准

### 90-100 分: 优秀 🌟
- 所有必需元素完整
- 链接网络密集
- 知识结构均衡

### 75-89 分: 良好 ✅
- 少量非关键问题
- 整体结构合理
- 需要小优化

### 60-74 分: 一般 ⚠️
- 存在一些需要修复的问题
- 知识结构有待完善
- 建议优先处理错误

### <60 分: 需要改进 ❌
- 多个关键问题
- 知识结构不完整
- 建议系统性优化
