# 知识库连接分析 Skill

## 功能
分析知识库中所有页面和概念之间的关联关系，发现潜在的知识连接，建议新的文章主题。

## 使用场景
当用户说以下任意一句话时触发：
- "分析连接"
- "发现关联"
- "suggest connections"
- "找出知识关联"
- "分析知识库结构"

## 执行步骤

### 1. 加载知识库索引
```bash
# 读取索引文件
cat knowledge-base/wiki/INDEX.json
```

### 2. 分析所有 Wiki 页面
读取 `knowledge-base/wiki/articles/` 目录下的所有页面：
- 提取每篇文章的标签
- 提取每篇文章的概念
- 提取每篇文章的分类
- 分析内容中的关键词

### 3. 构建知识图谱
创建一个概念关联矩阵：

```
概念A --[强关联]--> 概念B
  ├── 共同出现在: [文章1, 文章2]
  └── 关联度: 0.85

概念C --[弱关联]--> 概念D
  ├── 共同出现在: [文章3]
  └── 关联度: 0.45
```

### 4. 发现潜在连接
分析以下类型的关联：

#### 显式关联
- 相同标签的文章
- 相同概念的文章
- 相同分类的文章

#### 隐式关联
- 内容相似但未建立链接的文章
- 互补性知识（一个讲理论，一个讲实践）
- 层次关系（基础概念 -> 高级应用）

#### 时间关联
- 同一时期的文档
- 知识演进路径

### 5. 生成连接建议
输出格式：

```markdown
# 知识库连接分析报告

## 强关联（建议添加双向链接）

### 1. 机器学习 <-> 深度学习
- 关联度: 95%
- 共同概念: 神经网络、训练、模型
- 相关文章: 
  - 机器学习基础.md
  - 深度学习入门.md
- 建议: 在这两篇文章间添加双向链接

### 2. 前端框架 <-> React 教程
- 关联度: 88%
- 共同概念: 组件化、状态管理
- 建议: 创建综合文章"现代前端框架对比"

## 潜在新主题

### 1. 机器学习实践指南
- 原因: 有 3 篇文章分别讲理论、工具、案例，但缺少实践整合
- 相关文档:
  - 机器学习理论.md
  - ML工具对比.md
  - 实战案例.md
- 建议: 编译一篇综合实践指南

### 2. AI 发展史
- 原因: 多篇文章涉及不同时期的 AI 发展
- 建议: 创建时间线文章

## 知识空白

### 1. 缺少对比分析类文章
- 发现: 有多个相关概念但缺少对比
- 建议创建:
  - CNN vs RNN 对比
  - 不同前端框架对比

### 2. 缺少入门指南
- 发现: 某些高级主题缺少入门引导
- 建议: 为复杂主题创建"入门指南"

## 结构优化建议

1. 为"机器学习"相关文章添加统一标签
2. 在"深度学习"文章中引用"神经网络基础"
3. 创建知识图谱可视化
```

### 6. 自动执行优化（可选）
如果用户同意，自动执行：
- 在相关文章间添加双向链接
- 更新标签系统
- 创建缺失的基础文章框架

### 7. 更新索引
将发现的关联信息更新到 `INDEX.json`：

```json
{
  "connections": [
    {
      "page1": "机器学习基础.md",
      "page2": "深度学习入门.md",
      "strength": 0.95,
      "type": "strong"
    }
  ],
  "suggestions": ["建议列表"],
  "lastAnalyzed": "时间"
}
```

## 分析维度

### 内容相似度
- 关键词重叠
- 主题相似性
- 概念共现

### 知识结构
- 前置知识关系
- 并行知识关系
- 进阶知识关系

### 实用性
- 理论 + 实践配对
- 问题 + 解决方案配对
- 概念 + 案例配对
