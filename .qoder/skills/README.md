# 知识库管理 Skills

这是一组用于管理和维护个人知识库的 AI Skills。当你（AI Agent）需要处理知识库时，使用这些技能。

## 可用 Skills

### 1. 📄 文档分析 (kb-analyze-docs.md)
**功能**: 读取 raw 目录的原始文档，分析并生成 Wiki 页面

**触发词**:
- "分析文档"
- "处理文档"
- "导入文档"
- "ingest documents"
- "分析 raw 目录"

**做什么**:
- 扫描 `knowledge-base/raw/` 中的新文档
- 使用 LLM 分析每篇文档，提取摘要、标签、概念
- 生成结构化的 Wiki 页面到 `knowledge-base/wiki/articles/`
- 更新知识库索引

---

### 2. 🔨 知识库编译 (kb-compile-wiki.md)
**功能**: 整合相关文档，生成综合性知识文章

**触发词**:
- "编译知识库"
- "整合文档"
- "compile wiki"
- "生成综合文章"
- "整理知识库"

**做什么**:
- 识别可以整合的相关文档
- 按主题/概念分组
- 生成综合性 Wiki 文章
- 建立文档间的双向链接

---

### 3. 🔗 连接分析 (kb-analyze-connections.md)
**功能**: 分析知识库中的概念关联，发现潜在连接

**触发词**:
- "分析连接"
- "发现关联"
- "suggest connections"
- "找出知识关联"
- "分析知识库结构"

**做什么**:
- 分析所有 Wiki 页面的关联关系
- 构建知识图谱
- 发现强关联和弱关联
- 建议新的文章主题
- 识别知识空白

---

### 4. 🏥 健康检查 (kb-health-check.md)
**功能**: 全面检查知识库质量，发现问题并给出建议

**触发词**:
- "健康检查"
- "检查知识库"
- "lint wiki"
- "质量检查"
- "知识库体检"

**做什么**:
- 检查内容完整性（摘要、标签、链接等）
- 验证链接有效性（断裂链接、孤立页面）
- 检查内容质量（重复、过时、格式）
- 分析知识结构平衡性
- 生成评分和改进建议
- 可选：自动修复问题

---

## 典型工作流

### 新用户入门
```
1. 添加文档到 knowledge-base/raw/
2. 运行 "分析文档" → 生成初始 Wiki 页面
3. 运行 "编译知识库" → 整合相关文档
4. 在浏览器中查看结果
```

### 定期维护
```
1. 添加新文档到 raw/
2. 运行 "分析文档" → 处理新文档
3. 运行 "编译知识库" → 整合新内容
4. 运行 "健康检查" → 确保质量
5. 运行 "分析连接" → 发现新的关联
```

### 知识库优化
```
1. 运行 "健康检查" → 找出问题
2. 根据建议修复问题
3. 运行 "分析连接" → 发现缺失内容
4. 补充缺失的知识
5. 再次运行 "健康检查" → 确认改善
```

## 知识库目录结构

```
knowledge-base/
├── raw/                    # 原始文档目录
│   ├── article1.md
│   ├── article2.md
│   └── ...
│
└── wiki/                   # 生成的知识库
    ├── INDEX.json          # 知识库索引
    ├── articles/           # Wiki 文章
    │   ├── 主题1.md
    │   ├── 主题2.md
    │   └── ...
    └── concepts/           # 概念定义（可选）
        └── ...
```

## 使用示例

### 示例 1: 处理新文档
**用户**: "我往 raw 目录放了几篇新文章，帮我处理一下"

**AI**: 
1. 加载 `kb-analyze-docs.md` skill
2. 扫描 raw 目录的新文档
3. 逐个分析并生成 Wiki 页面
4. 报告处理结果

### 示例 2: 优化知识库
**用户**: "帮我检查一下知识库的质量"

**AI**:
1. 加载 `kb-health-check.md` skill
2. 全面检查知识库
3. 生成健康报告
4. 提供修复建议

### 示例 3: 发现知识关联
**用户**: "分析一下我的知识库，看看有什么新的发现"

**AI**:
1. 加载 `kb-analyze-connections.md` skill
2. 分析所有页面的关联
3. 发现潜在连接
4. 建议新的文章主题

## 注意事项

1. **增量处理**: 所有操作都应该支持增量，避免重复处理
2. **中文优先**: 生成的内容使用中文
3. **保持备份**: 重要操作前建议备份
4. **用户确认**: 大规模修改前先展示计划，获得用户同意
5. **错误处理**: 遇到错误要记录并继续处理其他内容
6. **可逆操作**: 尽量保证操作可逆，方便回滚
