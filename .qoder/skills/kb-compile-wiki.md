# 知识库编译 Skill

## 功能
整合 `knowledge-base/raw/` 目录中的相关文档，生成综合性的 Wiki 文章，建立概念之间的关联关系。

## 使用场景
当用户说以下任意一句话时触发：
- "编译知识库"
- "整合文档"
- "compile wiki"
- "生成综合文章"
- "整理知识库"

## 执行步骤

### 1. 分析现有文档
```bash
# 查看 raw 目录所有文档
ls knowledge-base/raw/

# 查看已有 Wiki 页面
ls knowledge-base/wiki/articles/
```

### 2. 识别概念关联
读取所有文档内容，找出：
- 共同的主题和概念
- 可以整合的文档组
- 缺失的综合视角

### 3. 分组相关文档
按概念或主题将文档分组，例如：
- 所有关于"机器学习"的文档
- 所有关于"前端开发"的文档
- 所有关于"项目管理"的文档

### 4. 生成综合 Wiki 页面
对每组相关文档，创建一篇综合性文章：

文件位置：`knowledge-base/wiki/articles/{主题}.md`

内容结构：
```markdown
---
title: {主题名称}
summary: {综合摘要}
tags: [综合, 主题相关标签]
concepts: [涉及的概念]
type: compiled
sources: [来源文档列表]
created: {当前时间}
---

# {主题名称}

## 概述

{对该主题的整体性介绍，200-300字}

## 核心概念

### 概念1
{概念解释，引用相关文档内容}

### 概念2
{概念解释，引用相关文档内容}

## 知识整合

{综合多个文档的观点，形成系统性认识}

## 实践应用

{如何在实际中应用这些知识}

## 相关资源

### 原始文档
- [[文档1]]
- [[文档2]]
- [[文档3]]

### 延伸阅读
- [[相关主题1]]
- [[相关主题2]]
```

### 5. 建立反向链接
检查新生成的页面与其他 Wiki 页面的关联：
- 在相关页面中添加指向新页面的链接
- 更新已有页面的"相关文章"部分

### 6. 更新索引
更新 `knowledge-base/wiki/INDEX.json`，添加新编译的页面信息。

### 7. 输出结果
向用户报告：
- 编译了多少个主题
- 生成了哪些综合文章
- 建立了哪些新的关联

## 智能整合策略

### 何时创建综合页面
- 同一主题有 2+ 个文档时
- 多个文档讨论相关概念时
- 发现知识空白需要补充时

### 整合原则
1. **系统性**：从零散到系统，形成知识体系
2. **深度**：不仅汇总，还要深化理解
3. **关联**：建立概念间的桥梁
4. **实用**：强调知识的实际应用

### 发现知识空白
如果某个主题只有浅层覆盖，标记为需要补充：
```markdown
> 💡 待补充: 关于 XXX 的内容还需要更多资料
```

